# 安全策略

## 支持的版本

当前仓库以 `main` 上最新提交为活跃维护线；发布标签（若使用）会在 Release 说明中列出。

## 报告漏洞

若你发现**可利用的安全漏洞**（例如未授权访问他人会话、密钥泄露路径、可远程利用的注入等），请**不要**在公开 Issue 中披露细节。

建议方式（任选其一）：

1. 使用 GitHub **Private vulnerability reporting**（若仓库已开启）。
2. 向维护者发送私信说明（通过你在用的协作渠道）。

请在报告中尽量包含：复现步骤、影响范围、相关版本或提交哈希。

## 非漏洞类问题

以下内容更适合走普通 Issue：

- LLM 输出内容不当（产品/提示词层面，而非代码漏洞）
- 依赖版本建议、性能优化

## 本地 HTTP 服务说明

`npm run http:interact` 提供的 HTTP 接口**默认绑定 127.0.0.1**，且无鉴权，仅用于本地联调。**切勿**在未加反向代理与认证的情况下直接暴露到公网。
