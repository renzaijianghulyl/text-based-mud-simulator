# 小程序类目与上架策略（决策树）

本文供 Chief 定案；具体规则以 [微信公众平台](https://mp.weixin.qq.com) 当时文档为准。

## 1. 是否具备「网络游戏」版号及相关资质？

- **是**：可走「游戏 > 角色扮演」等对应子类目，按官方清单准备资质、软著、版号材料与截图。
- **否**：进入第 2 步。

## 2. 核心交互是否为「纯文字阅读 + 用户输入生成文本」且无对战排行、充值抽卡等强游戏特征？

- **是**：评估「工具 / 信息查询」或「教育 / 人文」等弱游戏类目；强调历史教育、互动叙事属性；类目描述须与真实功能一致，避免虚假宣传。
- **否 / 不确定**：暂停上架时间表，先法务或平台咨询后再选类目。

## 3. 通用节点（所有类目）

- 提供《用户服务 / 隐私政策》链接：说明 openid、云开发、会话与叙事文本存储范围。
- 未成年人与内容合规：仅公有领域历史题材，不涉及架空或敏感现代题材（与产品原则一致）。
- 预留至少一次「审核驳回 → 改文案/改截图/改类目」的时间窗口。
